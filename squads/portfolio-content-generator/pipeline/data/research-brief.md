# Research Brief: Portfolio Copy para Desenvolvedores

## Fonte e Data

Pesquisa compilada durante a fase de discovery do squad Portfolio Content Generator.
Fontes: DEV Community, Pesto, Nucamp, WeAreDevelopers, Medium.

---

## Framework Principal: STAR para Descrições de Projeto

**S**ituation: contexto do problema ou oportunidade
**T**ask: qual era o papel do dev naquele contexto
**A**ction: o que foi construído ou decidido (com foco em decisões técnicas)
**R**esult: impacto mensurável ou qualitativo

Exemplo de aplicação:
> "O portifólio precisava ser bilíngue mas sem adicionar dependências pesadas. Implementei um sistema de i18n customizado via arquivos JSON, eliminando a necessidade do i18next e reduzindo o bundle em ~15kb."

---

## Framework Secundário: Elevator Pitch para Bio

`[Nome] + [Especialidade] + [Quem você ajuda/o que você resolve] + [Diferencial]`

Exemplo forte:
> "Fullstack engineer especializado em developer tooling. Transformo processos de 23 etapas em 4 comandos. Se você precisa de alguém que entrega e pensa em quem usa o software, bora conversar."

---

## Framework PAS para Bio (alternativo)

**P**roblem: o recrutador precisa de alguém que entregue, não de um lista de skills
**A**gitate: a maioria dos portifólios entrega tecnologias, não resultados
**S**olution: Diego entrega resultados — aqui está a evidência

---

## Sinais de Escrita Júnior vs. Sênior

| Júnior | Sênior |
|--------|--------|
| Lista tecnologias sem contexto | Descreve decisões arquiteturais |
| "I used React to build..." | "Escolhi React sobre Vue porque a complexidade de estado..." |
| Foco no processo | Foco no impacto |
| Projetos de tutorial clonados | Problemas reais resolvidos |
| "user-friendly", "robust", "scalable" | Métricas concretas e trade-offs |
| "passionate developer" | "Especialista em [domínio específico]" |

---

## Rubrica de Avaliação de Recrutadores

Recrutadores decidem em menos de 30 segundos se continuam lendo.

| Critério | Peso | Aprova | Reprova |
|----------|------|--------|---------|
| Clareza da proposta de valor | 25% | Bio em 2 frases que diz quem é e o que entrega | Bio genérica ou inexistente |
| Evidência de impacto | 25% | Métricas, antes/depois, decisões técnicas | "I built a web app" |
| Tom correto | 20% | Alinhado com a vaga/cultura | Tom genérico de template |
| Ausência de sinais de júnior | 15% | Raciocínio técnico visível | Apenas lista de features |
| Escaneabilidade | 15% | Parágrafos curtos, hierarquia clara | Blocos densos de texto |

## Vocabulário Profissional vs. Amador

| Amador usa | Profissional usa |
|------------|-----------------|
| passionate, hardworking | impact-driven, systems thinker |
| user-friendly | reduced friction, improved adoption |
| I built | I architected / I led / I shipped |
| technologies used | stack decision rationale |
| I like coding | I specialize in [domínio específico] |
| made it faster | reduced latency from Xms to Yms |
| simples, básico | não qualificar — descrever com precisão |

---

## Anti-Patterns mais Comuns (pesquisa com 200+ portifólios)

1. **Tech dump sem contexto** — listar tecnologias sem explicar o problema resolvido
2. **Paixão vazia** — "passionate developer who loves creating user-friendly applications"
3. **Foco no dev, não no avaliador** — escrever para outro programador quando quem lê é hiring manager
4. **Sem resultado** — descrever o que foi feito, nunca o impacto
5. **Clone de Tutorial** — todo-list, weather app sem nenhum diferencial
6. **Demo Quebrada** — links quebrados, telas em branco
7. **Quantidade sobre qualidade** — 12 projetos pela metade vs. 4 polidos
8. **Adjetivos sem evidência** — "high-performance" sem nenhum benchmark

## Exemplos de Copy Forte

### Bio forte (adaptada para perfil fullstack/game dev)

> "Engenheiro fullstack com queda por Game Dev. Construo desde APIs performáticas até lógica de jogo — pathfinding, state machines, física. Se você precisa de alguém que entende o código de ponta a ponta e ainda pensa na experiência de quem joga, é por aqui."

### Descrição de projeto forte

> "Precisava de internacionalização no portifólio mas não queria inflar o bundle com i18next. Criei um sistema de i18n via arquivos JSON com troca dinâmica de idioma. Resultado: PT-BR e EN funcionando, zero dependência adicional, bundle 15kb mais leve."
