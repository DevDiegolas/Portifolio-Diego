## Exemplo 2: Descrição de Projeto (Framework STAR)

### Projeto: Portifólio Pessoal com i18n Customizado

> **O problema:** precisava de um portifólio bilíngue (PT-BR/EN) sem inflar o bundle com bibliotecas de internacionalização.
>
> **O que fiz:** implementei um sistema de i18n leve usando arquivos JSON e Context API do React. Troca de idioma dinâmica, zero dependência de terceiros, e fácil de estender para novos idiomas.
>
> **Stack:** React 18 + TypeScript + Vite + Tailwind CSS
>
> **Resultado:** portifólio bilíngue funcional com bundle ~15kb menor do que a alternativa com i18next.

## Exemplo 4: Descrição de Jogo (Game Dev)

### Projeto: Jogo de Estratégia em Grid

> Jogo de estratégia baseado em turnos com pathfinding A* implementado do zero — sem biblioteca de path.
>
> O desafio foi garantir performance em grids grandes (100x100+) sem travar o frame. Resolvi com Web Workers para rodar o algoritmo fora da main thread, mantendo o jogo responsivo mesmo em maps complexos.
>
> **Stack:** TypeScript + Canvas API + Web Workers
>
> Cada unidade tem sua própria state machine (idle → moving → attacking → dead). Isso tornou a lógica de AI muito mais previsível de debugar.
